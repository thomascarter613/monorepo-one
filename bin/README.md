    # Executable Entry Points

    ## Purpose

    Small repository-level executable wrappers and command entry points intended to be run directly by developers or automation.

    ## Repository Role

    This directory is part of the canonical Monorepo One root-level structure.

    ## Rules

    Prefer thin wrappers here. Substantial implementation should live in tools, packages, or services.

    ## Current Status

    This directory is established as part of the root repository contract.

    Concrete implementation files will be added in later slices when the relevant capability is introduced.
