# Open Questions

1. What is the final repository name?
2. Is the project intended to be open-source, private, commercial, or dual-license?
3. What package scope should be used, if any?

## Tooling

1. Should Bun be the primary package manager/runtime?
2. Should Turbo and Moon both be used, and if so, what are their separate responsibilities?
3. Should Biome be the only formatter/linter at first?
4. Should Lefthook be used for Git hooks?

## Architecture

1. What is the first product domain this monorepo will support?
2. What apps, services, and packages should exist first?
3. What architectural boundaries are mandatory from day one?

## Operations

1. What deployment target should be assumed first?
2. Should Docker be added immediately or after the first runnable app?
3. What CI provider should be treated as canonical?

## AI / Handoff

1. What files should future AI assistants read first?
2. What project state should be machine-validated?
3. Should a repo contract checker be added early?
