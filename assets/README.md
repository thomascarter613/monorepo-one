    # Assets

    ## Purpose

    Shared non-code assets such as brand assets, design exports, diagrams, icons, images, and static media that are intentionally versioned.

    ## Repository Role

    This directory is part of the canonical Monorepo One root-level structure.

    ## Rules

    Large binary assets should be added deliberately. Generated/transient assets should go under .artifacts or an ignored build output directory.

    ## Current Status

    This directory is established as part of the root repository contract.

    Concrete implementation files will be added in later slices when the relevant capability is introduced.
