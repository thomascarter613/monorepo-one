    # Applications

    ## Purpose

    User-facing deployable applications, such as web apps, mobile apps, desktop apps, documentation sites, admin consoles, and portals.

    ## Repository Role

    This directory is part of the canonical Monorepo One root-level structure.

    ## Rules

    Apps may depend on packages and contracts. Apps should not contain reusable domain libraries that belong in packages.

    ## Current Status

    This directory is established as part of the root repository contract.

    Concrete implementation files will be added in later slices when the relevant capability is introduced.
